Recherche du mot "obertura"

ÉTAPE 1 : Constitution du corpus

  • 07, DEC 2025
  • POSTED BY MANON
  • Obertura

J'ai cherché les 50 urls dont j'avais besoin pour le projet. J'ai d'abord eu du mal à trouver, car le mot "obertura" (ouverture en catalan) existe aussi en espagnol, en basque et en occitan, et j'ai commencé par beaucoup tomber sur des urls dans ces langues. J'ai donc réfléchi aux différents sens du mot ouverture et j'ai ainsi pu affiner ma recherche, en ajoutant des mots-clés en catalan, liés aux différents sens du mot. J'ai trouvé 50 urls, que j'ai enregistrés dans un fichier txt pour pouvoir les utiliser facilement dans mon script bash. J'ai ensuite testé le script pour créer le tableau, et j'ai constaté que le script tournait pour la majorité des urls, mais qu'il s'arrêtait à la lecture de certains. J'ai donc dû retirer 4 urls pour ensuite en retrouver 4 fonctionnels.

ÉTAPE 2 : Création du tableau

  • 26, DEC 2025
  • POSTED BY MANON
  • Obertura

Lors des dernières séances de PPE, j'ai commencé le script bash permettant la création de mon tableau. J'ai d'abord créé le "squelette" du tableau en html, puis j'ai récupéré les informations à mettre dans le tableau avec un script bash. Un fois les informations (à savoir, le code http renvoyé par chaque url et l'encodage de chaque page) récupérées, j'ai rempli le tableau html (grâce à la commande `echo`, qui permet d'écrire dans un fichier html directement depuis le script bash). À ce stade (26/12/25), le tableau ne contenait donc que 4 colonnes (numéro de ligne, url, code http et encodage). J'ai donc ajouté 4 nouvelles colonnes: tout d'abord, le nombre de mots dans chaque page et le nombre d'occurrences du mot recherché ("obertura"). Pour cela j'ai utilisé les commandes `lynx` et `wc -w` pour compter le nombre total de mots, et `grep` et `wc -w` pour le nombre d'occurrences du mot recherché. J'ai constaté que pour certains urls, le nombre d'occurrences du mot "obertura" était de 0, même si le mot était bien présent au moins une fois dans la page. J'en déduis que les commandes utilisées sont sensibles à la casse, puisque dans certaines pages pour lesquelles le nombre d'occurrences est à 0, le mot n'apparaît qu'en majuscules/commençant par une majuscule. J'ai donc rajouté l'option `-i` à la commande `grep`. J'ai ensuite rajouté les dumps textuels et les aspirations (obtenus respectivement avec les commandes `dump` et `lynx`).

ÉTAPE 3 : Création du concordancier

  • 08, JAN 2025
  • POSTED BY MANON
  • Obertura

J'ai rajouté à mon script bash la création, en html, d'un concordancier. Un concordancier est un outil de référence, pour les linguistes, qui sert à rendre compte des différents contextes d'utilisation d'un mot dans un corpus donné (ici les dumps des 50 urls). En html, le concordancier se présente sous la forme d'un tableau à 3 colonnes dont la colonne centrale est le mot et les colonnes de gauche et de droite sont ses contextes gauche et droit (c'est-à-dire les mots ou lignes qui le précèdent et le suivent, dans chaque page, pour chaque occurrence du mot). Pour créer ce concordancier on ne part pas directement de l'url, mais du contenu des dumps textuels récupérés plus tôt. J'ai d'abord cherché toutes les occurrences du mot dans chaque dump, grâce à la commande `grep`, et j'ai utilisé ses options `-B` `-A` pour avoir ses contextes gauche et droit (`-B` permet de récupérer les lignes qui précèdent le mot, `-B 1` si on ne veut qu'une seule ligne, et `-A` permet de récupérer les lignes qui suivent le mot). J'ai stocké toutes les lignes trouvées dans un fichier temporaire (que j'ai appelé "contextes_temporaires_catalan.txt"), et j'ai ainsi pu procéder ligne par ligne en utilisant une boucle `while` (`while read -r line`), pour insérer chaque contexte dans le tableau. J'ai récupéré les contextes à l'aide des commande `echo` et `sed`, séparées par des pipes. J'ai ensuite rempli le concordancier, toujours dans la boucle `while`, puis j'ai fermé le tableau avec des balises html fermantes, et j'ai supprimé le fichier temporaire, à l'aide de `rm -f`. Enfin, j'ai ajouté une colonne "concordancier" au tableau, qui permet d'accéder directement au concordancier depuis le tableau.

ÉTAPE 4 : Préparation du texte pour PALS

  • 11, JAN 2025
  • POSTED BY MANON
  • Obertura

nous avons fait le script make_pals_corpus.sh, dont le but est de formater nos corpus pour les rendre lisibles par PALs (des scripts python d'analyse textométrique). Pour cela, il fallait d'abord tokeniser les textes (on travaille sur les contenus des dumps et sur les contextes), ce que j'ai fait avec la librairie nltk sur python. Le but étant d'avoir un script commun aux 4 membres du groupe, le script prenait 2 arguments: le dossier et la langue, pour que chacun puisse l'appliquer aux dumps et contextes de sa langue. Le script regroupait les contenus de tous les fichiers dumps dans un fichier dumps-[langue].txt et les fichiers contextes dans un fichier contexte-[langue].txt, avec comme format un mot par ligne, pour que les fichiers soient lisible par PALs. J'ai ensuite testé les scripts PALs sur mes fichiers dumps-catalan.txt et contextes-catalan.txt, ce qui m'a donné une liste des cooccurrents les plus fréquents du mot "obertura", dans les 50 urls de mon corpus. J'ai pu analyser les sens de ce mot qui ressortaient le plus en catalan. J'ai constaté que le mot était utilisé de façon presque identique au français, ce qui souligne la proximité des deux langues.
