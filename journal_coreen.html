Recherche du mot 개방 et de ses variantes 개방적, 개방성

ÉTAPE 1 : Constitution du corpus

  • 09, JAN 2025
  • POSTED BY LÉNA
  • 개방, 개방적, 개방성

Le mot ouverture en coréen est “개방”, mais il peut aussi être trouvé dans certains contextes “개방적” ou “개방성” pour ses autres significations. “개방” porte la notion d’ouverture au sens littéral alors que “개방적” et “개방성” désignent le caractère ouvert. Associés à certains noms comme “personnalité” : “개방적인 성격”, ils peuvent désigner le caractère ouvert d’esprit d’une personne. Pour constituer mon corpus, j’ai alors recherché sur internet le mot “개방” et ses variantes “개방적” et “개방성” pour récupérer les pages parlant de ces termes. J’ai pensé qu’il était aussi important de prendre des exemples concrets de la façon dont ces termes étaient généralement employés, j’ai donc cherché des articles de presse coréens qui contenaient au moins une occurrence. Je me suis notamment appuyée sur le corpus Korean News 2022 de la collection Leipzig Corpora qui permet de chercher un motif dans une multitude d’articles.

ÉTAPE 2 : Création du tableau

  • 10, JAN 2025
  • POSTED BY LÉNA
  • 개방, 개방적, 개방성

Pour créer le tableau des urls récupérées, j’ai d’abord écrit un code html avec les colonnes qui doivent apparaître dedans c’est-à-dire le numéro de ligne, le lien de la page, son code HTTP, son encodage, le nombre de mots sur la page, le nombre d’occurrences de “개방”, “개방적” ou “개방성”, le lien vers les contextes autour des mots et le lien vers le concordancier. Je suis ensuite passée à l’écriture du script bash en créant des variables pour le fichier de sortie html, pour les chemins de chacun des dossiers, et pour un compteur de lignes. Je récupère la page avec cURL que je stock dans mon dossier aspirations et la réponse donnée dans une autre variable afin de la réutiliser pour extraire le code HTTP et l’encodage. Pour le code HTTP, j’ai rajouté une condition pour que la page ne soit pas traitée si le code est différent de 200, et une ligne au tableau qui indique dans chaque colonne qu’aucune donnée n’a été extraite. Après avoir récupéré l’encodage, je peux également rajouter une condition pour convertir les encodages différents de UTF-8 vers UTF-8 afin que toutes les pages aient le même encodage lors du traitement. J’utilise ensuite la commande lynx pour stocker le texte brut de la page dans le dossier dumps-text. Je peux ensuite placer mes variables à l’intérieur du code html du tableau et d’incrémenter le compteur dans la boucle pour que tous les fichiers et le tableau soient générés correctement.

ÉTAPE 3 : Création du concordancier

  • 11, JAN 2025
  • POSTED BY LÉNA
  • 개방, 개방적, 개방성

Cette étape m’a causée le plus de difficultés car, dans un premier temps, le coréen n’est pas tokénisé de la même façon que le français à cause de sa nature agglutinante, et dans un second temps, ce n’est pas un mot mais 3 variantes que je cherche à extraire dans le concordancier. Tout d’abord pour la tokénisation, j’ai installé PykoSpacing spécialement conçu pour créer des espaces entre les mots coréens grâce à la commande pykos, puis je supprime les lignes vides avec la commande grep -v '^[[:space:]]*$' avant de rediriger le résultat vers un fichier temporaire. Pour ensuite extraire mon mot et ses variantes, j’ai écrit un script awk qui ajoute “개방”, “개방적” ou “개방성” dans la colonne mot de mon concordancier selon ce qui a été trouvé dans la ligne, et redirige le fichier vers le chemin de le dossier concordance, que j’ai stocké dans une variable. Je fais la même chose pour extraire les contextes dans un fichier texte mais cette fois sans ajouter les balises html dans les commandes print, en redirigeant cette voir vers ma variable qui contient le chemin vers le dossier contextes.

ÉTAPE 4 : Préparation du texte pour PALS

  • 11, JAN 2025
  • POSTED BY LÉNA
  • 개방, 개방적, 개방성

Pour préparer mon texte au traitement par les PALS, j’ai créé un script qui prend comme arguments un dossier qui contient tous les textes à traiter et la langue des textes. Je forme une boucle que chaque fichier de mon dossier soit traité. Si la langue rentrée est “coreen”, les textes passent par une première commande : pykos, afin d’espacer les mots coréens, puis par la commande sed, qui dans un premier temps entoure tous les signes de ponctuation d’un retour à la ligne, et dans un second temps remplace les espaces par un retour à la ligne. Enfin, la commande tr -s '\n' supprime les sauts de ligne en trop afin d’avoir en sortie un seul fichier texte qui combine les mots de tous les fichiers du dossier ensemble et de façon continue. Désormais, pour que ce script s’applique autant aux fichiers dumps-text qu’aux fichiers contextes, j’ai créé une condition qui change le contenu de la variable output_file selon le dossier passer en argument, afin que les fichiers en sortie soient dans le bon dossier et portent le nom correspondant.
